# 고객 이탈 예측 및 이탈 전 증상 분석 프로젝트

---

## 1. 프로젝트 개요

본 프로젝트는 Telco 고객 데이터를 활용해 고객 이탈 여부를 예측하고, 이탈에 영향을 주는 핵심 요인을 도출하며, 단순 예측을 넘어 이탈 이전에 나타나는 위험 신호를 식별하는 것을 목표로 수행했다. 전체 과정은 BigQuery SQL 기반 데이터 정제 및 피처 설계, Colab 기반 EDA와 머신러닝 모델링, 모델 해석과 비즈니스 전략 도출의 흐름으로 진행했다.

본 프로젝트의 핵심 방향은 다음과 같다.

- SQL 단계에서 전처리와 파생변수 생성을 최대한 수행해 분석 효율을 높임
- 단순히 예측 정확도만 높이는 것이 아니라, 운영 가능한 모델 구조를 검토함
- 예측 결과를 바탕으로 실제 리텐션 전략에 연결 가능한 인사이트를 도출함

## 2. 데이터 구조 및 분석 대상

고객 데이터는 Customer를 중심으로 다음과 같은 테이블 구조를 가진다.

```
Customer ─┬─ Demographics
          ├─ Services
          ├─ Status
          ├─ Location
          └─ Population
```

이를 고객 단위로 조인하여 분석용 마트를 구성했다. 주요 변수는 다음 범주로 구분된다.

| 범주 | 주요 변수 |
|------|-----------|
| 인구통계 | 성별, 연령, 시니어 여부, 결혼 여부, 부양가족 여부 |
| 서비스 정보 | 가입기간, 인터넷 유형, 계약 형태, 결제 방식, 월 요금, 총 요금, 총 매출 |
| 상태 정보 | 만족도, 이탈 여부, 이탈 점수, CLTV |
| 지역 정보 | 도시, 우편번호, 위도/경도, 인구 규모 |

---

## 3. BigQuery SQL 기반 데이터 전처리

분석 1단계에서는 BigQuery SQL을 사용해 고객 단위 통합 테이블 `analysis_base`를 생성했다. 이 과정에서 단순 `SELECT *`가 아니라 실제 분석 가능한 형태로 컬럼을 정제하는 데 집중했다.

주요 작업은 다음과 같다.

- 공백이 포함된 컬럼명을 백틱으로 처리
- `Total Charges` 등 수치형 컬럼의 타입 정제
- 고객 단위 조인 구조 설계
- 이후 분석과 모델링이 가능한 형태로 별칭 통일

이 과정에서 `SAFE_CAST`, `COALESCE` 등을 활용해 BigQuery에서 직접 정제 가능한 구조를 만들었다. 이를 통해 Colab에 데이터를 불러온 이후에는 무거운 전처리보다 분석과 모델링에 집중할 수 있도록 구성했다.

## 4. SQL 기반 피처 엔지니어링

`analysis_base`를 바탕으로 `feature_mart`를 생성하며 파생변수를 설계했다. 단순 원본 컬럼만 사용하는 대신, 이탈과 관련이 있을 가능성이 높은 변수들을 SQL에서 미리 생성했다.

생성한 대표 파생변수는 다음과 같다.

| 파생변수 | 설명 |
|----------|------|
| `TenureYearGroup` | 가입기간을 연 단위로 그룹화 |
| `TenureBand` | 가입기간 구간 레이블 (0-5m, 6-11m, …) |
| `IsLongTermCustomer` | 장기 고객 여부 플래그 |
| `ChargeToMonthlyRatio` | 총 요금 대비 월 요금 비율 |
| `AvgChargePerMonth` | 고객당 월 평균 요금 |
| `RevenueToCltvRatio` | 총 매출 대비 CLTV 비율 |
| `MonthlyChargeBand` | 월 요금 구간 레이블 |
| `SatisfactionBand` | 만족도 구간 레이블 |
| `PopulationBand` | 거주 지역 인구 규모 구간 |
| `ContractRiskLevel` | 계약 형태 기반 위험도 레이블 |

이 단계의 목적은 단순 데이터 적재가 아니라, "이탈과 어떤 관계가 있을까"라는 분석 관점을 변수 설계에 반영하는 것이었다.

## 5. SQL 기반 1차 인사이트 도출

모델링 전 SQL로 직접 주요 변수별 이탈률을 집계해 1차 인사이트를 확인했다.

| 계약 형태 | 이탈률 |
|-----------|--------|
| Month-to-Month | **45.84%** |
| One Year | 10.71% |
| Two Year | 2.55% |

### 인터넷 유형별 이탈률

| 인터넷 유형 | 이탈률 |
|-------------|--------|
| Fiber Optic | **40.72%** |
| Cable | 25.66% |

| None | 7.40% |

### 가입기간 구간별 이탈률

| 가입기간 구간 | 이탈률 |
|---------------|--------|
| 0–5개월 | **54.71%** |
| 6–11개월 | 35.97% |
| 12–23개월 | 29.51% |
| 24–35개월 | 22.03% |
| 36–47개월 | 19.52% |
| 48개월 이상 | 9.64% |

이 결과를 통해 다음과 같은 패턴을 확인했다.

- 단기 계약 고객의 이탈률이 매우 높음
- 가입 초기 고객의 이탈 위험이 집중됨
- Fiber Optic 고객군의 이탈률이 높음
- 장기 가입자일수록 이탈 위험이 감소함

즉, 계약 형태, 가입기간, 인터넷 유형이 이탈 설명의 핵심 축임을 초기 단계에서 확인할 수 있었다.

## 10. PCA 기반 차원 축소 실험

선별 피처 기반 모델에 대해 PCA를 적용한 추가 차원 축소 실험도 수행했다.

### 차원 축소 결과

| 구분 | Shape | 설명분산 유지율 |
|------|-------|----------------|
| Before PCA | (7043, 29) | — |
| After PCA | (5634, 19) | **95.3%** |

즉, 원핫인코딩 후 29차원 데이터를 설명분산 95.3%를 유지하면서 19개 주성분으로 줄였다.

이후 XGBoost로 재학습한 결과는 다음과 같다.

| 모델 | 사용 변수 | F1-score | ROC-AUC | 해석 |
|------|-----------|----------|---------|------|
| XGBoost PCA | 선별 피처 + PCA (29→19) | 0.658 | 0.875 | 차원 축소는 되었지만 성능 하락 |

### XGBoost PCA — Classification Report

```
              precision    recall  f1-score   support

         0.0       0.89      0.84      0.86      1035
         1.0       0.61      0.71      0.66       374

    accuracy                           0.80      1409
   macro avg       0.75      0.77      0.76      1409
weighted avg       0.82      0.80      0.81      1409
```

### 과적합 점검

```
Train F1:      0.892    |    Test F1:      0.658
Train ROC-AUC: 0.990    |    Test ROC-AUC: 0.875
```

**해석**

- 차원 축소 자체는 성공했으나 성능 향상에는 기여하지 못함
- 트리 기반 모델인 XGBoost는 PCA로 압축된 주성분보다 원래의 해석 가능한 피처 공간에서 더 잘 작동할 가능성이 큼
- 이번 데이터에서는 PCA가 효율성 개선보다 정보 손실과 일반화 저하를 가져온 것으로 판단됨

## 11. 클래스 불균형 보정 실험

이탈 고객은 정상 고객보다 적은 클래스 불균형 구조를 가지므로, XGBoost FS에 대해 SMOTE 기반 오버샘플링 실험을 수행했다. 오버샘플링은 학습 데이터에만 적용했다.

### SMOTE 적용 전후 학습 데이터 분포

| 클래스 | Before SMOTE | After SMOTE |
|--------|-------------|------------|
| 비이탈 (0) | 4,139명 | 4,139명 |
| 이탈 (1) | 1,495명 | **4,139명** |

결과는 다음과 같다.

| 모델 | 사용 변수 | F1-score | ROC-AUC | 해석 |
|------|-----------|----------|---------|------|
| XGBoost + SMOTE | 선별 피처 + SMOTE | 0.662 | 0.880 | 불균형 보정 실험이었으나 기존 FS 대비 성능 개선 없음 |

### XGBoost + SMOTE — Classification Report

```
              precision    recall  f1-score   support

           0       0.89      0.85      0.87      1035
           1       0.63      0.70      0.66       374

    accuracy                           0.81      1409
   macro avg       0.76      0.77      0.77      1409
weighted avg       0.82      0.81      0.81      1409
```

**해석**

- 기존 XGBoost FS보다 F1-score와 ROC-AUC가 모두 하락
- 현재 데이터에서는 `scale_pos_weight`만 반영한 기본 학습 방식이 더 적절했을 가능성이 큼
- 인위적 오버샘플링이 오히려 분포 왜곡을 유발했을 수 있음

## 12. 이탈 전 증상 분석

### 12-2. 이탈 위험 세그먼트 룰 도출

Contract × TenureBand × InternetType 조합별 실제 이탈률과 평균 위험도를 산출했다.

| Contract | TenureBand | InternetType | 고객수 | 평균 예측위험도 | 실제 이탈률 |
|----------|-----------|--------------|--------|---------------|------------|

이 결과를 통해 다음과 같은 이탈 전 증상을 도출할 수 있었다.

- **Month-to-Month 계약**은 가장 강한 위험 신호다.
- 가입 초기 구간, 특히 **0~11개월**에서 이탈 위험이 집중된다.

- 대표적인 고위험 시나리오는 **Month-to-Month + 0~5개월 + Fiber Optic** 조합이다.

즉, 본 프로젝트는 이탈 여부 예측뿐 아니라, "어떤 조건일 때 이탈 가능성이 높아지는가"라는 사전 증상 분석까지 확장했다는 점에서 의미가 있다.

## 13. SHAP 분석 — XGBoost FS 모델 기준

> SHAP(SHapley Additive exPlanations)을 활용해 모델의 예측 결과를 피처 단위로 해석했다. 단순 변수 중요도를 넘어, 각 피처가 이탈 위험을 높이는지 낮추는지 방향성까지 파악할 수 있었다.

```
분석 대상: XGBoost FS 모델
SHAP 값 행렬: (1409, 29)   ← 테스트셋 고객 수 × 피처 수
```

---

### 13-1. 전체 변수 중요도 해석 (Summary Plot + Bar Plot)

SHAP `summary plot`과 `bar plot`을 종합하면, 주요 변수 중요도 순위는 다음과 같다.

| 순위 | 피처 | 역할 | 방향성 |
|------|------|------|--------|
| 1 | `Contract_Two Year` | 계약 안정성 | 이탈 위험 ↓ (보호 요인) |
| 2 | `Dependents` | 생활 기반 안정성 | 이탈 위험 ↓ (보호 요인) |
| 3 | `ChargeToMonthlyRatio` | 과금 누적 비율 | 낮을수록 이탈 위험 ↑ |
| 4 | `Contract_One Year` | 계약 안정성 | 이탈 위험 ↓ (보호 요인) |
| 5 | `MonthlyCharge` | 월 요금 부담 | 높을수록 이탈 위험 ↑ |
| 6 | `Age` | 연령 | 일부 구간에서 이탈 위험 ↑ |
| 7 | `RevenueToCltvRatio` | 고객 가치 대비 매출 | 높을수록 이탈 위험 ↑ |
| 8 | `AvgChargePerMonth` | 월 평균 과금 | 높을수록 이탈 위험 ↑ |
| 9 | `TenureInMonths` | 가입기간 | 짧을수록 이탈 위험 ↑ |

이 결과는 모델이 고객 이탈 여부를 판단할 때 가장 우선적으로 보는 축이 **계약 형태**라는 점을 분명하게 보여준다. 특히 `Contract_Two Year`는 단일 변수 중 가장 큰 영향력을 가졌으며, 이는 계약 안정성이 고객 유지에 가장 강한 설명력을 가진다는 의미다.

또한 `ChargeToMonthlyRatio`, `RevenueToCltvRatio`, `AvgChargePerMonth`와 같은 파생변수들이 상위 중요도에 위치한 점은 SQL 단계에서 설계한 피처 엔지니어링이 실제 설명력 향상에 기여했음을 보여준다.

---

### 13-2. 이탈 위험 방향성 해석 (Summary Plot)

`summary plot`을 통해 각 피처의 중요도뿐 아니라 방향성까지 확인할 수 있었다.

**보호 요인 (이탈 위험 ↓)**

| 피처 | 조건 | 해석 |
|------|------|------|
| `Contract_Two Year` | 값 = 1 (2년 약정 가입) | SHAP 값이 크게 음수로 이동 → 가장 강력한 유지 신호 |
| `Contract_One Year` | 값 = 1 (1년 약정 가입) | 이탈 위험 감소, 단 Two Year보다 약함 |
| `Dependents` | 값 = 1 (부양가족 있음) | 생활 안정성이 이탈 억제로 작용 |
| `TenureInMonths` | 값이 클수록 | 가입기간이 길수록 이탈 위험 감소 |
| `ChargeToMonthlyRatio` | 값이 클수록 | 과금 관계가 누적될수록 안정성 ↑ |

**위험 요인 (이탈 위험 ↑)**

| 피처 | 조건 | 해석 |
|------|------|------|
| `MonthlyCharge` | 값이 높을수록 | 과금 부담 증가 → 이탈 가능성 ↑ |
| `AvgChargePerMonth` | 값이 높을수록 | 월 평균 부담과 이탈 위험 정비례 |
| `RevenueToCltvRatio` | 값이 높을수록 | CLTV 대비 과도한 매출 추출 신호 |
| `TenureInMonths` | 값이 작을수록 | 가입 초기 고객 이탈 위험 집중 |
| `InternetType_Fiber Optic` | 값 = 1 | Fiber Optic 고객의 이탈 위험 상대적으로 높음 |

### 13-3. 개별 고객 예측 해석 (Force Plot)

`force plot` 분석에서는 개별 고객 1명의 예측 결과를 기준으로, 어떤 요인이 이탈 위험을 높였고 어떤 요인이 낮췄는지 직접 확인할 수 있었다.

**분석 사례 (idx = 0)**

```
실제값:   유지
예측값:   유지
이탈 확률: 0.211
```

| 구분 | 요인 | 작용 방향 |
|------|------|-----------|
| 위험 요인 | `Contract_One Year = 0`, `Contract_Two Year = 0` | 장기계약 보호 효과 없음 → 이탈 위험 ↑ |
| 보호 요인 | `Dependents = 1` | 생활 기반 안정 → 이탈 위험 ↓ |
| 보호 요인 | `PaymentMethod_Credit Card = 1` | 안정적 결제 방식 → 이탈 위험 ↓ |
| 보호 요인 | `TenureInMonths = 48` | 장기 가입 이력 → 이탈 위험 ↓ |
| 보호 요인 | `InternetType_DSL = 1` | Fiber Optic 대비 낮은 위험 → 이탈 위험 ↓ |
| 보호 요인 | `AvgChargePerMonth`, `RevenueToCltvRatio` | 과금 수준 안정 → 이탈 위험 ↓ |

이 고객은 계약 구조만 보면 다소 불안정한 요소가 있었지만, 장기 가입 이력, 안정적인 생활 기반, 결제 방식 등 복합적 보호 요인으로 인해 최종적으로 낮은 이탈 위험으로 예측되었다. **동일하게 단기 계약 구조를 가진 고객이라도, 가입기간과 고객 속성에 따라 실제 이탈 가능성이 달라질 수 있음**을 보여주는 사례다.

### 13-4. Dependence Plot — 피처 값에 따른 위험도 변화 패턴

`dependence plot`을 통해 특정 피처 값의 변화가 SHAP 기여도에 어떤 영향을 미치는지 확인했다.

| 피처 | 관찰된 패턴 | 실무 해석 |
|------|-------------|-----------|
| `TenureInMonths` | 가입기간 짧을수록 SHAP ↑, 60개월 이후 강하게 음수 | 초기 고객 이탈 리스크 집중, 장기 고객은 매우 안정 |
| `MonthlyCharge` | 요금 높을수록 SHAP 양수로 상승 | 높은 월 요금 → 실질적 이탈 위험 증가 |
| `ChargeToMonthlyRatio` | 값 작을수록 위험 높고, 클수록 SHAP 감소 | 가입 초기 / 누적 관계 부족 고객일수록 불안정 |
| `InternetType_Fiber Optic` | 1일 때 SHAP 양수, 0일 때 상대적으로 낮음 | Fiber Optic 고객의 이탈 위험이 전반적으로 높음 |
| `Contract_Two Year` | 1일 때 SHAP 강하게 음수 | 2년 약정은 이탈 억제 효과가 가장 명확한 보호 요인 |

---

### 13-5. SHAP 분석의 실무적 의미

SHAP 분석은 단순히 "어떤 변수가 중요하다"는 수준을 넘어, **각 피처가 이탈 위험을 높이는지 낮추는지를 방향성까지 포함해 해석**할 수 있게 해주었다.

| SHAP 결과 | 연결 가능한 비즈니스 전략 |
|-----------|--------------------------|
| `Contract_Two Year`, `Contract_One Year` 보호 효과 | 장기계약 전환 캠페인 근거 |
| `MonthlyCharge`, `AvgChargePerMonth` 위험 요인 | 요금 부담 완화 / 할인 프로모션 설계 |
| `TenureInMonths`, `ChargeToMonthlyRatio` 초기 위험 | 가입 초기 온보딩 강화 / 조기 이탈 방지 캠페인 |
| `InternetType_Fiber Optic` 위험 요인 | Fiber Optic 고객 서비스 품질 점검 / 만족도 조사 |
| `Dependents` 보호 요인 | 가족 단위 고객 대상 번들 상품 강화 |

즉 SHAP는 이번 프로젝트에서 모델을 단순 예측 도구가 아니라 **설명 가능한 의사결정 지원 도구**로 전환하는 역할을 했다. 이를 통해 "왜 이 고객이 이탈 위험군인가"를 구체적으로 설명하고, 고객군별로 차별화된 리텐션 전략을 설계할 수 있는 기반을 마련했다.

## 15. 최종 결론

- 실제 운영 환경에서 적용 가능한 입력 변수 중심 구성

따라서 본 프로젝트의 **최종 운영형 모델은 XGBoost FS**로 판단했다.
