# 고객 이탈 예측 및 이탈 전 증상 분석 프로젝트

## 1. 프로젝트 개요

본 프로젝트는 Telco 고객 데이터를 활용해 고객 이탈 여부를 예측하고, 이탈에 영향을 주는 핵심 요인을 도출하며, 단순 예측을 넘어 이탈 이전에 나타나는 위험 신호를 식별하는 것을 목표로 수행했다. 전체 과정은 BigQuery SQL 기반 데이터 정제 및 피처 설계, Colab 기반 EDA와 머신러닝 모델링, 모델 해석과 비즈니스 전략 도출의 흐름으로 진행했다.

본 프로젝트의 핵심 방향은 다음과 같다.

- SQL 단계에서 전처리와 파생변수 생성을 최대한 수행해 분석 효율을 높임
- 단순히 예측 정확도만 높이는 것이 아니라, 운영 가능한 모델 구조를 검토함
- 예측 결과를 바탕으로 실제 리텐션 전략에 연결 가능한 인사이트를 도출함

---

## 2. 데이터 구조 및 분석 대상

고객 데이터는 `Customer`를 중심으로 다음과 같은 테이블 구조를 가진다.

- `Customer`
- `Demographics`
- `Services`
- `Status`
- `Location`
- `Population`

이를 고객 단위로 조인하여 분석용 마트를 구성했다. 주요 변수는 다음 범주로 구분된다.

- 인구통계: 성별, 연령, 시니어 여부, 결혼 여부, 부양가족 여부
- 서비스 정보: 가입기간, 인터넷 유형, 계약 형태, 결제 방식, 월 요금, 총 요금, 총 매출
- 상태 정보: 만족도, 이탈 여부, 이탈 점수, CLTV
- 지역 정보: 도시, 우편번호, 위도/경도, 인구 규모

---

## 3. BigQuery SQL 기반 데이터 전처리

분석 1단계에서는 BigQuery SQL을 사용해 고객 단위 통합 테이블 `analysis_base`를 생성했다. 이 과정에서 단순 `SELECT *`가 아니라 실제 분석 가능한 형태로 컬럼을 정제하는 데 집중했다.

주요 작업은 다음과 같다.

- 공백이 포함된 컬럼명을 백틱으로 처리
- `Total Charges` 등 수치형 컬럼의 타입 정제
- 고객 단위 조인 구조 설계
- 이후 분석과 모델링이 가능한 형태로 별칭 통일

이 과정에서 `SAFE_CAST`, `COALESCE` 등을 활용해 BigQuery에서 직접 정제 가능한 구조를 만들었다. 이를 통해 Colab에 데이터를 불러온 이후에는 무거운 전처리보다 분석과 모델링에 집중할 수 있도록 구성했다.

## 4. SQL 기반 피처 엔지니어링

`analysis_base`를 바탕으로 `feature_mart`를 생성하며 파생변수를 설계했다. 단순 원본 컬럼만 사용하는 대신, 이탈과 관련이 있을 가능성이 높은 변수들을 SQL에서 미리 생성했다.

생성한 대표 파생변수는 다음과 같다.

- `TenureYearGroup`
- `TenureBand`
- `IsLongTermCustomer`
- `ChargeToMonthlyRatio`
- `AvgChargePerMonth`
- `RevenueToCltvRatio`
- `MonthlyChargeBand`
- `SatisfactionBand`
- `PopulationBand`
- `ContractRiskLevel`

이 단계의 목적은 단순 데이터 적재가 아니라, "이탈과 어떤 관계가 있을까"라는 분석 관점을 변수 설계에 반영하는 것이었다.

## 5. SQL 기반 1차 인사이트 도출

모델링 전 SQL로 직접 주요 변수별 이탈률을 집계해 1차 인사이트를 확인했다.

### 계약 형태별 이탈률
- `Month-to-Month`: 45.84%
- `One Year`: 10.71%
- `Two Year`: 2.55%

### 인터넷 유형별 이탈률
- `Fiber Optic`: 40.72%
- `Cable`: 25.66%
- `DSL`: 18.58%
- `None`: 7.40%

### 가입기간 구간별 이탈률
- `0-5개월`: 54.71%
- `6-11개월`: 35.97%
- `12-23개월`: 29.51%
- `24-35개월`: 22.03%
- `36-47개월`: 19.52%
- `48개월 이상`: 9.64%

이 결과를 통해 다음과 같은 패턴을 확인했다.

- 단기 계약 고객의 이탈률이 매우 높음
- 가입 초기 고객의 이탈 위험이 집중됨
- `Fiber Optic` 고객군의 이탈률이 높음
- 장기 가입자일수록 이탈 위험이 감소함

즉, 계약 형태, 가입기간, 인터넷 유형이 이탈 설명의 핵심 축임을 초기 단계에서 확인할 수 있었다.

## 6. Colab 기반 모델링 환경 구축

BigQuery에서 구성한 `feature_mart`를 `pandas-gbq`를 통해 Colab으로 불러왔다. 이후 범주형 변수 인코딩, 학습/테스트 데이터 분할, 모델 학습 및 평가를 수행했다.

사용 모델은 다음과 같다.

- Random Forest
- XGBoost

또한 이탈 클래스의 불균형을 고려해 `class_weight` 또는 `scale_pos_weight`를 반영해 학습을 진행했다.

## 7. 전체 변수 기반 모델 성능

전체 변수 기반 모델의 결과는 다음과 같다.

| 모델 | 사용 변수 | F1-score | ROC-AUC | 해석 |
|---|---|---:|---:|---|
| Random Forest | 전체 변수 기반 | 0.843 | 0.963 | 성능이 우수하고 기준 모델로 활용 가능 |
| XGBoost | 전체 변수 기반 | 0.920 | 0.990 | 최고 성능 기록, 다만 만족도 계열 변수 영향이 매우 큼 |

Random Forest는 비교적 안정적인 성능을 보였고, XGBoost는 가장 높은 성능을 기록했다. 그러나 XGBoost 중요도 결과에서 `SatisfactionScore`, `SatisfactionBand`가 상위에 집중되었다.

이는 두 가지 가능성을 시사한다.

- 만족도 계열 변수가 실제로 매우 강한 설명력을 가질 수 있음
- 또는 이 변수가 타깃과 지나치게 가까운 정보라 데이터 누수 가능성이 있을 수 있음

따라서 최고 성능 결과를 그대로 최종 모델로 채택하기보다, 운영 가능성을 고려한 추가 검증이 필요했다.

## 8. 누수 의심 변수 제거 모델 검증

만족도 계열과 이탈 이후 정보일 가능성이 있는 컬럼을 제거한 `XGBoost v2`를 별도로 학습했다.

### 제외한 주요 컬럼
- `ChurnLabel`
- `ChurnCategory`
- `ChurnReason`
- `CustomerStatus`
- `ChurnScore`
- `SatisfactionScore`
- `SatisfactionBand`

결과는 다음과 같다.

| 모델 | 사용 변수 | F1-score | ROC-AUC | 해석 |
|---|---|---:|---:|---|
| XGBoost v2 | 누수 의심 변수 제거 | 0.692 | 0.893 | 사전 확보 가능한 정보만으로도 유의미한 예측력 유지 |

이 결과는 전체 변수 기반 모델보다 성능은 하락했지만, 누수 가능성을 낮춘 상태에서도 계약 형태, 가입기간, 인터넷 유형, 과금 패턴 등만으로 충분히 예측력이 유지된다는 점을 보여주었다.

즉, 초기 초고성능 모델은 일부 타깃 근접 변수에 의존했을 수 있으며, 실무 적용 관점에서는 `v2`와 같은 보수적 모델 검증이 필요하다는 결론을 얻었다.

## 9. 피처 선택 기반 경량 모델 구축

초기 모델은 원핫인코딩 이후 컬럼 수가 증가해 학습 효율과 해석성 측면에서 불리할 수 있었다. 따라서 변수 중요도와 도메인 해석을 바탕으로 핵심 피처만 선별한 `XGBoost FS` 모델을 구축했다.

### 선별 피처
- `Age`
- `SeniorCitizen`
- `Married`
- `Dependents`
- `TenureInMonths`
- `TenureBand`
- `Offer`
- `PhoneService`
- `InternetType`
- `Contract`
- `PaymentMethod`
- `MonthlyCharge`
- `ChargeToMonthlyRatio`
- `AvgChargePerMonth`
- `PopulationBand`
- `RevenueToCltvRatio`
- `ChurnValue`

이 방식은 엄밀히 말하면 피처 엔지니어링보다는 피처 선택에 가깝지만, 프로젝트 측면에서는 중복 변수와 고차원 변수를 제거해 더 실용적인 입력 구조를 만든 과정으로 볼 수 있다.

결과는 다음과 같다.

| 모델 | 사용 변수 | F1-score | ROC-AUC | 해석 |
|---|---|---:|---:|---|
| XGBoost FS | 선별 피처 기반 | 0.697 | 0.893 | 핵심 변수만으로 유사 성능 확보, 운영형 모델로 적합 |

### 혼동행렬 해석
- 실제 비이탈 1035명 중 852명을 정확히 분류
- 실제 이탈 374명 중 298명을 정확히 분류
- 이탈 클래스 recall은 0.80

즉, 고위험 고객을 넓게 포착하는 용도로는 충분히 활용 가능한 성능이었다.

### 과적합 점검 결과
- Train F1: 0.762
- Test F1: 0.697
- Train ROC-AUC: 0.938
- Test ROC-AUC: 0.893

완전히 차이가 없는 것은 아니지만, 과도한 과적합이라기보다는 현실적인 수준의 일반화 성능으로 해석할 수 있었다.

## 10. PCA 기반 차원 축소 실험

선별 피처 기반 모델에 대해 PCA를 적용한 추가 차원 축소 실험도 수행했다.

### 전처리 결과
- Before PCA shape: `(7043, 29)`
- After PCA shape: `(5634, 19)`
- Explained variance ratio sum: `0.9532`

즉, 원핫인코딩 후 29차원 데이터를 설명분산 95.3%를 유지하면서 19개 주성분으로 줄였다.

이후 XGBoost로 재학습한 결과는 다음과 같다.

| 모델 | 사용 변수 | F1-score | ROC-AUC | 해석 |
|---|---|---:|---:|---|
| XGBoost PCA | 선별 피처 + PCA(29→19) | 0.658 | 0.875 | 차원 축소는 되었지만 성능 하락 |

### 과적합 점검 결과
- Train F1: 0.892
- Test F1: 0.658
- Train ROC-AUC: 0.990
- Test ROC-AUC: 0.875

### 해석
- 차원 축소 자체는 성공했으나 성능 향상에는 기여하지 못함
- 트리 기반 모델인 XGBoost는 PCA로 압축된 주성분보다 원래의 해석 가능한 피처 공간에서 더 잘 작동할 가능성이 큼
- 이번 데이터에서는 PCA가 효율성 개선보다 정보 손실과 일반화 저하를 가져온 것으로 판단됨

## 11. 클래스 불균형 보정 실험

이탈 고객은 정상 고객보다 적은 클래스 불균형 구조를 가지므로, `XGBoost FS`에 대해 `SMOTE` 기반 오버샘플링 실험을 수행했다. 오버샘플링은 학습 데이터에만 적용했다.

결과는 다음과 같다.

| 모델 | 사용 변수 | F1-score | ROC-AUC | 해석 |
|---|---|---:|---:|---|
| XGBoost + SMOTE | 선별 피처 + SMOTE | 0.662 | 0.880 | 불균형 보정 실험이었으나 기존 FS 대비 성능 개선 없음 |

### 해석
- 기존 `XGBoost FS`보다 F1-score와 ROC-AUC가 모두 하락
- 현재 데이터에서는 `scale_pos_weight`만 반영한 기본 학습 방식이 더 적절했을 가능성이 큼
- 인위적 오버샘플링이 오히려 분포 왜곡을 유발했을 수 있음

## 12. 이탈 전 증상 분석

본 프로젝트는 단순히 "이탈할지 말지" 예측하는 데 그치지 않고, 이탈 이전에 어떤 조건이 위험 신호로 나타나는지 분석했다. 이를 위해 `XGBoost FS` 모델의 예측확률을 기반으로 고위험 고객군을 추출하고, 계약 형태, 가입기간, 인터넷 유형 조합별 평균 위험도와 실제 이탈률을 비교했다.

### 주요 고위험 조합

| Contract | TenureBand | InternetType | 고객수 | 평균 예측위험도 | 실제 이탈률 |
|---|---|---|---:|---:|---:|
| Month-to-Month | 0-5m | Fiber Optic | 96 | 0.873 | 0.740 |
| Month-to-Month | 6-11m | Fiber Optic | 41 | 0.827 | 0.707 |
| Month-to-Month | 0-5m | Cable | 42 | 0.709 | 0.643 |
| Month-to-Month | 0-5m | DSL | 70 | 0.590 | 0.614 |
| Month-to-Month | 24-35m | Fiber Optic | 60 | 0.620 | 0.583 |

이 결과를 통해 다음과 같은 이탈 전 증상을 도출할 수 있었다.

1. `Month-to-Month` 계약은 가장 강한 위험 신호다.
2. 가입 초기 구간, 특히 `0~11개월`에서 이탈 위험이 집중된다.
3. 같은 조건이라면 `Fiber Optic` 고객의 위험이 더 높다.
4. 단일 변수보다 `계약 형태 + 가입기간 + 인터넷 유형`의 조합이 더 강한 설명력을 가진다.
5. 대표적인 고위험 시나리오는 `Month-to-Month + 0~5개월 + Fiber Optic` 조합이다.

즉, 본 프로젝트는 이탈 여부 예측뿐 아니라, "어떤 조건일 때 이탈 가능성이 높아지는가"라는 사전 증상 분석까지 확장했다는 점에서 의미가 있다.

## 14. 최종 결론

본 프로젝트에서 가장 높은 성능은 전체 변수 기반 XGBoost가 기록했지만, 만족도 계열 변수와 이탈 이후 정보일 수 있는 컬럼이 성능에 크게 기여했기 때문에 실무 적용 모델로는 해석에 주의가 필요했다.

반면 `XGBoost FS`는 다음 장점을 가졌다.

- 누수 가능성을 낮춘 구조
- 핵심 변수만으로도 유의미한 예측력 유지
- 해석 가능성이 높음
- 실제 운영 환경에서 적용 가능한 입력 변수 중심 구성

따라서 본 프로젝트의 최종 운영형 모델은 `XGBoost FS`로 판단했다.

---

## 15. 비즈니스 제안

분석 결과를 바탕으로 다음과 같은 실행 전략을 제안할 수 있다.

- `Month-to-Month` 고객 대상 장기계약 전환 프로모션 제공
- 가입 후 `0~6개월` 고객 대상 온보딩 및 유지관리 캠페인 운영
- `Fiber Optic` 고객군에 대해 가격 민감도와 품질 이슈를 집중 점검
- `단기 계약 + 초기 가입기간 + Fiber Optic` 조합을 최우선 관리군으로 설정
- 고위험군 고객에 대해 1개월, 3개월, 6개월 시점 선제적 접촉 전략 운영

## 16. 프로젝트 요약

- BigQuery SQL에서 고객 단위 분석 마트를 구축하고 전처리 및 파생변수 생성을 수행했다.
- Colab에서 Random Forest와 XGBoost 기반 이탈 예측 모델을 비교했다.

- PCA와 SMOTE 실험도 수행했으나, 이번 데이터에서는 성능 개선 효과가 제한적이었다.
- 최종적으로 계약 형태, 가입기간, 인터넷 유형, 과금 패턴이 고객 이탈의 핵심 설명 변수임을 확인했다.
- 단순 예측을 넘어, 이탈 전 증상과 고위험 조건 조합까지 도출해 실무적 활용 가능성을 제시했다.
